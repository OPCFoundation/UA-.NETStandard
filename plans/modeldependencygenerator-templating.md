# Plan: Refactor `ModelDependencyGenerator` to use the templating system

## Goal

Replace the direct `TextWriter.WriteLine`/`Write` calls in `ModelDependencyGenerator` with the project's `TemplateString` + `Template` infrastructure used by every other generator (e.g. `ConstantsGenerator`, `XmlSchemaGenerator`, `BinarySchemaGenerator`). The runtime output is byte-for-byte identical; the goal is consistency, header re-use, and unit-test parity.

## Why

- Every other emitter in `Tools/Opc.Ua.SourceGeneration.Core/Generators/` is template-driven. `ModelDependencyGenerator` is the only outlier and was written that way to ship the `nodeset-deps`/`design-deps` work quickly.
- Going through `Template` automatically gets us:
  - The shared `{{Tokens.CodeHeader}}` (license + tool/version banner) rendered the same way as every other `*.g.cs` file.
  - Consistent line trimming / indentation handling via `TemplateWriter`.
  - List iteration via `TemplateDefinition` (the `Targets` + `Load`/`Render` callback pattern) — much easier to extend when we need to filter/dedupe additional sources.
- Several `Tokens.*` we need (`Prefix`, `ModelUri`, `Version`) already exist; only one new token (`PublicationDate`) is needed.

## Files to change

### New
- `Tools/Opc.Ua.SourceGeneration.Core/Generators/ModelDependencyTemplates.cs`
  - `public static readonly TemplateString File` — shell with `{{Tokens.CodeHeader}}` + comment block + `{{Tokens.ListOfModelDependencies}}` (new token).
  - `public static readonly TemplateString Entry` — single line: `[assembly: global::Opc.Ua.ModelDependencyAttribute("{{Tokens.ModelUri}}", "{{Tokens.Prefix}}", {{Tokens.Version}}, {{Tokens.PublicationDate}})]`.
  - For nullable string args: emit them as the literal `null` or quoted string. The cleanest approach is to put quoted-string-or-null *into the replacement value itself* (a small helper in the generator does the quoting) and keep the template token-only — i.e. `{{Tokens.Version}}` is replaced with either `null` or `"1.05.04"`. This mirrors how `ConstantsTemplates` lets the caller decide string vs symbol.

### Modified
- `Tools/Opc.Ua.SourceGeneration.Core/Generators/ModelDependencyGenerator.cs`
  - Drop `EmitAttribute`, `Quote`, `FormatDate` raw-writer helpers. Keep `Quote` and `FormatDate` as private statics (still needed to pre-format replacement values).
  - Build a `List<EntryRow>` (private record struct) of unique `(ModelUri, Prefix, Version, PublicationDate)` tuples in dedupe order: self → declared deps → referenced-assembly closure (existing logic preserved).
  - `Emit()` opens the file via `m_context.FileSystem.CreateTextWriter`, wraps it in a `TemplateWriter`, builds a `Template` against `ModelDependencyTemplates.File`, calls `template.AddReplacement(Tokens.ListOfModelDependencies, ModelDependencyTemplates.Entry, entries, LoadEntry, WriteEntry)`, then `template.Render()`.
  - `LoadEntry` returns `context.TemplateString` (no per-row template variation needed).
  - `WriteEntry` adds `Tokens.ModelUri`, `Tokens.Prefix`, `Tokens.Version`, `Tokens.PublicationDate` and renders.
- `Tools/Opc.Ua.SourceGeneration.Core/Templating/Tokens.cs`
  - Add `public static string PublicationDate => nameof(PublicationDate);`.
  - Add `public static string ListOfModelDependencies => nameof(ListOfModelDependencies);`.

### Tests
- `Tests/Opc.Ua.SourceGeneration.Tests/ModelDependencyGeneratorTests.cs` (extend if exists, otherwise create)
  - Snapshot test: feed a synthetic `IGeneratorContext` with one self-namespace + one declared-dep + one referenced-only entry; assert the rendered file contains:
    - The `// <auto-generated />` header from `CodeTemplates.CodeHeader`.
    - Three `[assembly: global::Opc.Ua.ModelDependencyAttribute(...)]` lines in the documented order.
    - The OpcUa root namespace is **not** emitted.
    - A null `Version`/`PublicationDate` renders as bare `null`, not `"null"`.
  - Round-trip test: parse the emitted file with the existing `ModelDependencyScanner` (or `ReferenceMetadataScanner`) and assert all three entries come back equal to the inputs.
- Existing source-generator end-to-end tests should still pass byte-equivalent — confirm by running the full source-generator test suite before/after.

## Work breakdown

1. **`templates-and-tokens`** — Add `ModelDependencyTemplates.cs` + the two new tokens.
2. **`generator-refactor`** — Rewrite `ModelDependencyGenerator.Emit()` to use `Template`; keep `Quote`/`FormatDate` helpers; produce identical text output (verified by diffing against a captured baseline).
3. **`unit-tests`** — Add focused `ModelDependencyGeneratorTests.cs` snapshot + round-trip cases.
4. **`regression`** — Run the full source-generator + Opc.Ua.SourceGeneration.Tests suite to confirm no consumer (`*.ModelDependencies.g.cs` content) changed.

## Risks / open questions

- **Output drift.** `TemplateWriter` enforces some whitespace rules (trim trailing line-breaks, indentation pop). The current raw output ends with a single trailing newline after the last `]`; the template path will likely produce the same but could differ by one byte. Mitigation: capture a baseline before refactor and assert exact equality in the snapshot test.
- **Header banner change.** The current generator writes its own `// <auto-generated />` + 2 comment lines. The shared `Tokens.CodeHeader` is *more verbose* (license block). This **is** a visible output change but aligns with every other `*.g.cs` file, so it's the desired outcome. Document in commit message.
- **`Tokens.Version` collision.** `Template` constructor pre-populates `Tokens.Version` with the *tool* version. We must not use that token for the per-attribute `version` argument — use the new `Tokens.AssemblyVersion` token name? **Decision needed.** Cleanest: add a new `Tokens.ModelVersion` token rather than re-purposing `Version`. Updating the plan accordingly: use `Tokens.ModelVersion` and `Tokens.ModelPublicationDate` to avoid any chance of collision with the tool-banner replacements.
- **No behavioral change** to dedupe order, `IsValid` filter, OpcUa-root skip, or self-version fallback (`TargetVersion ?? target.Version`).

## Tokens (final)

| Token | Existing? | Purpose |
| --- | --- | --- |
| `Tokens.CodeHeader` | yes | shared license + tool banner |
| `Tokens.ListOfModelDependencies` | **new** | list slot for the per-attribute template |
| `Tokens.ModelUri` | yes | per-entry URI |
| `Tokens.Prefix` | yes | per-entry C# prefix |
| `Tokens.ModelVersion` | **new** | per-entry version (literal `null` or quoted string) |
| `Tokens.ModelPublicationDate` | **new** | per-entry publication date (literal `null` or quoted ISO 8601) |
